[백준] 7795번: 먹을 것인가 먹힐 것인가

Lee Hyun Kyeong on Dec 28, 20212021-12-28T00:17:04+09:00
Updated Jan 122022-01-12T22:46:49+09:00 6 min read

📖 문제 설명

심해에는 두 종류의 생명체 A와 B가 존재한다. A는 B를 먹는다. A는 자기보다 크기가 작은 먹이만 먹을 수 있다. 예를 들어, A의 크기가 {8, 1, 7, 3, 1}이고, B의 크기가 {3, 6, 1}인 경우에 A가 B를 먹을 수 있는 쌍의 개수는 7가지가 있다. 8-3, 8-6, 8-1, 7-3, 7-6, 7-1, 3-1.

두 생명체 A와 B의 크기가 주어졌을 때, A의 크기가 B보다 큰 쌍이 몇 개나 있는지 구하는 프로그램을 작성하시오.

💦 입력

첫째 줄에 테스트 케이스의 개수 T가 주어진다. 각 테스트 케이스의 첫째 줄에는 A의 수 N과 B의 수 M이 주어진다. 둘째 줄에는 A의 크기가 모두 주어지며, 셋째 줄에는 B의 크기가 모두 주어진다. 크기는 양의 정수이다. (1 ≤ N, M ≤ 20,000)

🍩 출력

각 테스트 케이스마다, A가 B보다 큰 쌍의 개수를 출력한다.

🤯 풀이 로직

  • 테스트 케이스 개수 입력
  • 테스크 케이스 개수만큼 반복
    • N과 M을 입력
    • 배열 A, B를 입력
    • A, B 정렬
    • A 순회
      • B 순회
        • 현재 A값이 B값보다 크면
          • B의 현재 인덱스를 정답에 추가
          • B 순회 중지
    • 정답 출력

문제의 예제 입력을 통해 풀이 로직을 풀어 써 보겠다. A와 B가 다음과 같다고 하자.

1
2

A : 8 1 7 3 1   
B : 3 6 1

A와 B를 내림차순 한다. (코드상에서는 오름차순 정렬하고 반복문을 뒤에서부터 돌렸다.)

1
2

A : 8 7 3 1 1
B : 6 3 1

이제 이중 for문을 통해 각 A의 원소들이 B의 생명체를 얼만큼 먹을 수 있는지 탐색한다.
A가 8일 때를 확인해보자. B의 처음인 6보다 8이 더 크다. 이미 B는 내림차순 정렬되어있으므로 6이후의 값은 볼 필요가 없다. 6을 먹었다는 것은 그 이후의 생명체, 즉 현재의 6보다 작은 값들이 보장된 생명체가 있으므로 다 먹을 수 있다는 의미이다.

필요한 자료형

  • 테스트 케이스 개수 : int t
  • a의 수 : int n
  • b의 수 : int m
  • a를 담을 배열 : int[] a = new int[n+1]
  • b를 담을 배열 : int[] b = new b[m+1]
  • 정답 : int ans

필요한 함수

  1. 정답을 찾을 함수
    • A 순회
      • B 순회
    • 현재 A값이 B값보다 크면
      • B의 현재 인덱스를 정답에 추가
      • B 순회 중지

💭 회고

원래는 내림차순 정렬해서 비교하려고 했으나, int 배열을 Comparable로 구현하려면 다시 Integer로 감싸야 한다는.. 지난날의 악몽이 떠올라서 현타올 뻔했으나.. 그냥 오름차순 정렬해 놓고 뒤에서부터 오면 되겠구나 생각이 들었다! 처음 알고리즘 시작했을 때는 무조건 코드부터 짜기 시작했는데 이제 풀기전에 어떤 변수, 자료구조를 쓸 건지 어떻게 풀어갈지 적어놓고 함수로 각각 분리해서 푸니까 더 깔끔해진 것 같다! 많이 나아졌다.

🌠 배운것

  • 간단한 int 배열 내림차순 같은 경우에는 Integer로 감싸서 Comparable을 구현하는 것보다 오름차순 정렬하고 반복문을 뒤에서부터 시작하면 된다는 것을 배웠다.

import java.io.*;
import java.util.*;

public class Main {
    static int t; // 테스트케이스 개수
    static int n; // A 개수
    static int m; // B 개수
    static int[] a; // A를 담을 배열
    static int[] b; // B를 담을 배열
    static int ans; // 정답

    static void input() throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        String[] tokens = br.readLine().split(" ");
        t = Integer.parseInt(tokens[0]);
        for (int i=0; i<t; i++) {
            tokens = br.readLine().split(" ");
            n = Integer.parseInt(tokens[0]);
            m = Integer.parseInt(tokens[1]);
            a = new int[n+1];
            b = new int[m+1];
            ans = 0;

            tokens = br.readLine().split(" ");
            for (int j=0; j<n; j++) {
                a[j] = Integer.parseInt(tokens[j]);
            }

            tokens = br.readLine().split(" ");
            for (int j=0; j<m; j++) {
                b[j] = Integer.parseInt(tokens[j]);
            }
            output();
        }
    }

    static void output() {
        Arrays.sort(a);
        Arrays.sort(b);

        for (int i=a.length-1; i > -1; i--) {
            for (int j=b.length-1; j > -1; j--) {
                if (a[i] > b[j]) {
                    ans += j;
                    break;
                }
            }
        }
        System.out.println(ans);
    }

    public static void main(String[] args) throws IOException {
        input();
    }

}
