## 코틀린 장점
---
- 긴결한 구문 작성
- `null safety` 지원으로 안전한 코드 작성
- 자바와 100% 호환되는 상호운용성
- 코루틴 기법을 통해 비동기 프로그래밍 간소화 (네트워크 연동/데이터베이스 갱신)


## 코틀린의 파일명
---
자바는 무조건 클래스로 묵어야하기 때문에 클래스명과 파일명이 같아야 한다. 하지만 코틀린은 파일명과 클래스명이 달라도 된다. 이 말은 곧 코틀린은 클래스로 묶지 않아도 된다는 의미이고, 최상위에 클래스뿐만 아니라 변수, 함수를 올려놓을 수가 있다는 의미이다.  
자바와 코틀린은 100% 호환이 된다고 했는데 만약 코틀린에서 최상위에 변수나 함수를 올려놓을 경우 자바로 어떻게 100% 호환이 된다는 의미일까. 
여기서 코틀린이 어떻게 바이트 코드를 만드는지 알아두어야 한다! 예를들어 `Test.kt`라는 파일이 있을 때, 이 파일의 최상위에 변수와 함수가 있고 Student 클래스가 있다고 해보자.
자바에서는 컴파일을 하면 Test.class 파일이 생기겠지만, 코틀린은 최상위 변수와 함수 따로, 최상위 클래스 따로 파일을 만들게 된다. 
최상위 변수와 함수는 `파일명+Kt.class` 파일로 만들고 최상위 클래스는 `클래스명.class` 파일로 만들게 된다.

## 변수 선언하기
---
<ins>val/var</ins> <ins>변수명</ins>: <ins>타입</ins> = <ins>값</ins>

|    키워드       |                설명                        |
|:---------------|:------------------------------------------:|
| val (value)    | 초기값이 할당되면 값을 바꿀 수 없는 변수 선언 |
| var (variable) | 초기값이 할당되면 값을 바꿀 수 없는 변수 선언 |


## 초기화 미루기
---
최상위에 선언한 변수와 클래스의 멤버변수는 선언과 동시에 초기값을 할당해야 한다. 그런데 변수 선언시 초기값을 할당할 수 없는 경우에는 다음과 같이 2가지 방식으로 초기화를 미룰 수 있다.
1. `lateinit` 키워드 사용
- `var` 키워드로 선언한 변수에만 사용할 수 있다.
- `Int`, `Long`, `Double`, `Float`, `Boolean`, `Byte` 타입에는 사용할 수 없다.

2. `lazy` 키워드 사용
- 변수 선언문 뒤에 `by lazy { }` 형식으로 선언하여 사용한다.
- 해당 변수가 최초로 사용될 때 중괄호 부분이 실행되어 초기값에 할당된다.

```kotlin
var num: Int by lazy { 10 }
fun main() {
    println(num + 100) // 이때 lazy의 중괄호가 실행되어 num이 10으로 초기화된다.
}
```

## 객체 타입
---
코틀린의 모든 변수는 객체이고, 모든 타입은 객체 타입이다.
- 기초 타입 객체 : `Int`, `Short`, `Long`, `Double`, `Float`, `Byte`, `Boolean` 
- 문자와 문자열 객체 : `Char`, `String` (삼중 따옴표(""")를 사용하면 입력한 줄 바꿈, 들여쓰기가 그대로 반영된다.)
- 모든 타입 가능한 객체 : `Any` (코틀린의 최상위 클래스)
- 반환문이 없는 함수의 객체 : `Unit` (특수한 상황을 표현하려는 목적으로 사용)
- null이나 예외를 반환하는 함수의 객체 : `Nothing` (특수한 상황을 표현하고 null만 대입할 수 있어서 데이터로서의 의미는 없다.)
- 널 허용과 불허용 : 타입뒤에 `물음표(?)`를 추가하면 널 허용
